# SRC vs DIST - Relationship Explained

## ЁЯФД THE CONNECTION

The `dist` folder is the **compiled output** of the `src` folder.

```
src/ (TypeScript Source)
  тЖУ
  тЖУ npm run build (TypeScript Compiler)
  тЖУ
dist/ (JavaScript Output)
```

## ЁЯУК COMPARISON

| Aspect | src/ | dist/ |
|--------|------|-------|
| **Language** | TypeScript (.ts) | JavaScript (.js) |
| **Purpose** | Source code (human-readable) | Compiled code (machine-readable) |
| **Editable** | тЬУ YES - Edit here | тЬЧ NO - Auto-generated |
| **Generated** | тЬЧ Manual creation | тЬУ Auto-generated by tsc |
| **Committed to Git** | тЬУ YES | тЬЧ NO (in .gitignore) |
| **Used in Production** | тЬЧ NO | тЬУ YES |
| **Used in Development** | тЬУ YES (with ts-node) | тЬЧ NO (use npm run dev) |

## ЁЯФД HOW THE COMPILATION WORKS

### Step 1: TypeScript Compiler Configuration (tsconfig.json)
```json
{
  "compilerOptions": {
    "rootDir": "./src",      // Read from src/
    "outDir": "./dist",      // Write to dist/
    "module": "esnext",      // Module format
    "target": "esnext"       // JavaScript version
  }
}
```

### Step 2: Build Command
```bash
npm run build
```
This runs: `tsc` (TypeScript Compiler)

### Step 3: Compilation Process
```
TypeScript (.ts) files in src/
    тЖУ
    тЖУ Type checking
    тЖУ Syntax validation
    тЖУ Code transformation
    тЖУ
JavaScript (.js) files in dist/
```

## ЁЯУЭ EXAMPLE: How src/server.ts Becomes dist/server.js

### src/server.ts (TypeScript)
```typescript
import express from "express";
import { pool } from "./config/db.js";

const app = express();
const PORT: number = process.env.PORT || 5000;

app.listen(PORT, () => {
  console.log(`Server running on port ${PORT}`);
});
```

### dist/server.js (JavaScript - After Compilation)
```javascript
import express from "express";
import { pool } from "./config/db.js";

const app = express();
const PORT = process.env.PORT || 5000;

app.listen(PORT, () => {
    console.log(`Server running on port ${PORT}`);
});
```

**Changes made by compiler:**
- Type annotations removed (`: number`)
- Comments preserved
- Syntax remains the same (ESM modules)
- Source maps generated (.js.map)
- Type declarations created (.d.ts)

## ЁЯФД WORKFLOW

### Development Workflow
```
1. Edit src/server.ts
   тЖУ
2. npm run dev (uses ts-node)
   тЖУ
3. ts-node compiles on-the-fly
   тЖУ
4. Server runs with hot reload
   тЖУ
5. No need to manually build
```

### Production Workflow
```
1. Edit src/server.ts
   тЖУ
2. npm run build
   тЖУ
3. TypeScript compiler generates dist/
   тЖУ
4. node dist/server.js
   тЖУ
5. Server runs from compiled JavaScript
```

## ЁЯОп KEY POINTS

### тЬУ DO THIS
- Edit files in `src/` directory
- Run `npm run build` after changes
- Commit `src/` to git
- Use `npm run dev` for development

### тЬЧ DON'T DO THIS
- Edit files in `dist/` directory (changes will be lost)
- Commit `dist/` to git (it's auto-generated)
- Delete `dist/` folder (it will be regenerated)
- Manually modify compiled JavaScript

## ЁЯФН GENERATED FILES IN dist/

### 1. .js Files (Compiled JavaScript)
```javascript
// dist/server.js
import express from "express";
// ... compiled code
```

### 2. .js.map Files (Source Maps)
```json
{
  "version": 3,
  "file": "server.js",
  "sourceRoot": "",
  "sources": ["../src/server.ts"],
  "mappings": "..."
}
```
**Purpose**: Maps compiled JavaScript back to original TypeScript for debugging

### 3. .d.ts Files (Type Declarations)
```typescript
// dist/server.d.ts
export declare const app: Express;
export declare const PORT: number;
```
**Purpose**: Provides TypeScript type information for consumers

## ЁЯЪА WHEN TO REBUILD

Rebuild `dist/` when:
- тЬУ You modify any file in `src/`
- тЬУ You add new files to `src/`
- тЬУ You delete files from `src/`
- тЬУ You change `tsconfig.json`
- тЬУ You're preparing for production

## ЁЯФз TROUBLESHOOTING

### Problem: Changes in src/ not reflected in dist/
**Solution**: Run `npm run build`

### Problem: dist/ folder is missing
**Solution**: Run `npm run build` to regenerate

### Problem: Module not found errors
**Solution**: 
1. Check file exists in `src/`
2. Verify import paths are correct
3. Run `npm run build`

### Problem: Old code still running
**Solution**: 
1. Stop the server
2. Run `npm run build`
3. Restart the server
